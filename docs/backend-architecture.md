## Стек и архитектура бэкенда TaxiVillage

### Выбор стека

- **Платформа**: Node.js + TypeScript.
- **Фреймворк**: **NestJS** — модульная архитектура, удобные DI, Guards/Interceptors, хорошая структура для роста проекта.
- **ORM**: **Prisma** поверх **PostgreSQL**:
  - типобезопасные запросы,
  - удобные миграции и генерация клиента,
  - хорошая поддержка отношений между сущностями.
- **Аутентификация и авторизация**:
  - JWT (access + refresh) с ролями `passenger`, `driver`, `admin`,
  - Guards в NestJS для проверки роли и доступа к маршрутам.
- **Реальное время**:
  - WebSocket‑шлюз (Socket.IO или встроенный WebSockets в NestJS) для:
    - обновления статуса поездок,
    - передачи координат водителей.

### Структура проекта `backend/` (план)

- `src/main.ts` — точка входа HTTP + WebSocket сервера.
- `src/app.module.ts` — корневой модуль NestJS.
- `src/config/` — конфигурация (переменные окружения, БД, JWT и т.д.).
- `src/prisma/` — модуль Prisma (сервис, модуль, исключения).
- `src/auth/` — аутентификация и авторизация (регистрация, логин, refresh, guards, роли).
- `src/users/` — общий модуль пользователей.
- `src/drivers/` — управление профилями водителей и их документами/статусом.
- `src/rides/` — логика заказов поездок, статусы и назначение водителей.
- `src/tariffs/` — тарифы и расчёт примерной стоимости.
- `prisma/schema.prisma` — схема БД с моделями `User`, `PassengerProfile`, `DriverProfile`, `Car`, `Ride`, `Tariff`, `RideStatusHistory`.

### Ключевые решения

- **NestJS + Prisma + PostgreSQL** позволяют быстро сделать прототип и при этом оставить запас по масштабируемости.
- **Один API для всех клиентов**:
  - мобильное приложение пассажира,
  - мобильное приложение водителя (тот же API, разные роли),
  - веб‑админка.
- **Расширяемость**:
  - можно добавить:
    - модули оплаты,
    - push‑уведомления,
    - систему рейтингов,
    - предварительные заказы,
    - аналитику.
